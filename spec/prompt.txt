Project Name: Forty-Nine (四十九蓍)
Core Concept: 这是一个严格遵循周易“大衍筮法”底层数理逻辑的起卦模拟器。拒绝一键生成，必须强制用户进行交互，通过反复的机械动作（18变）让心静下来，完成起卦过程。

1. UX/UI 视觉与交互规范

视觉风格： 极简、留白、东方禅意。背景使用宣纸色（#F5F5F5），主体元素为深墨色（#333333）。

核心元素： 页面中央显示由细线模拟的“蓍草”。

核心交互： 采用手势滑动/鼠标拖拽 (Drag/Swipe)。用户必须在横向的蓍草堆上划一刀，系统根据划动的 X 轴坐标比例，将蓍草在视觉上“分而为二”。绝对不能做成点击按钮！

动画要求： 所有的移动、分开、聚拢动作，必须平滑且缓慢（使用类似 transition: all 1.5s ease-in-out），营造从容不迫的仪式感。

2. 核心流程状态机 (State Flow)

State 0 (准备)： 展示 50 根蓍草。提示“大衍之数五十，请取一置于高阁”。用户点击/拖动一根至顶部固定（象征太极，不参与运算），剩余 49 根。

State 1-18 (十八变循环)： 每3变为一个循环（生成一爻），共6个循环（生成六爻）。

提示用户“分而为二”。用户划动屏幕，当前蓍草分为左右两堆。

动画演示算法扣除过程：右堆减1（挂一），左右堆分别以4为单位剔除（揲四），归拢余数和挂一（归奇），拿走。

展示本变剩余的蓍草数量，聚拢后进入下一变。

State 19 (成卦)： 18变结束，六爻自下而上排列完毕。界面淡出蓍草，浮现计算出的六个爻象。

3. 核心算法：“一变”的推演逻辑
每次用户触发划动分割时，执行以下计算：

输入： total (当前草数，初始为49) 和 splitRatio (用户划动位置的比例，0.1 - 0.9)。

步骤1 (象两)： leftPile = Math.floor(total * splitRatio); rightPile = total - leftPile;

步骤2 (挂一)： rightPile -= 1;

步骤3 (揲四归奇)：

leftRemainder = leftPile % 4 === 0 ? 4 : leftPile % 4;

rightRemainder = rightPile % 4 === 0 ? 4 : rightPile % 4;

步骤4 (计算剩余)： >     * 拿走的草数：removed = 1 + leftRemainder + rightRemainder;

进入下一变的草数：nextTotal = total - removed;

4. 三变成爻映射规则
每一爻经历“三变”后，最终剩余的 nextTotal 必定是 24, 28, 32, 或 36。
